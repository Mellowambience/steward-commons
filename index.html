Steward Commons

An open protocol for preserving how ideas, decisions, evidence, and outcomes evolve — so reasoning becomes a public good that can be reviewed, forked, audited, and improved.

● Draft v0.1 · model & OS agnostic · no lock-in

The reasoning graph

Every project becomes a living graph of judgment. The shape below is enforced by the JSON schemas in this repo — and by the validator further down.

flowchart LR P[Project] --> D[Decision] D --> P1[Proposal A] D --> P2[Proposal B] P1 --> E1[Evidence / Objection] P2 --> E2[Evidence / Objection] D -->|selected| S((Chosen)) D --> O[Outcome] O --> R[Reflection] R -.->|improves| D

Live validator

Paste a record (the JSON for a project, decision, proposal, evidence, outcome, or reflection) and validate it against the protocol schemas — entirely in your browser, no server.

What's in the repo

schemas/

6 JSON Schemas (draft 2020-12): project, decision, proposal, evidence, outcome, reflection.

steward.py

A zero-dependency CLI that runs the MVP and validates vaults against the schemas.

docs/

Protocol, architecture, data model, constitution, vision, glossary, API, CLI.

examples/

A sample project showing the reasoning graph in real data.

Run it locally

Clone the repo and try the reference implementation:

python steward.py demo  — builds + validates a full reasoning graph
python steward.py init && python steward.py project new --name "Atlas"  — start your own vault
python steward.py validate  — check every record against the schemas
